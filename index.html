Embracing Secure AI Agents

AI Agents are becoming a key platform for deploying LLM in real-world applications, and their technological advances make it possible to automate and make decisions for complex tasks. Their integration with critical systems (e.g., automated shopping, automated finance) brings new security risks—uncontrollable inputs, opaque logic, and dynamic environments expose them to attacks such as fraud or workflow hijacking. Related companies are in urgent need of finding vulnerabilities in Agent and proposing a strong security framework to ensure safe deployment. Therefore, we conducted the following research.

Research Foundations

Vulnerabilities in the Environment of AI Agent

Attackers can manipulate the Agent's environment (such as website images or knowledge bases) by generating adversarial samples through diffusion models, causing the Agent to perform malicious operations. However, although diffusion models are good at generating high-quality images, their potential as adversarial tools, especially in black-box attacks (without access to the target model or training data), has not been fully studied. In this paper, we investigate the adversarial capabilities of diffusion models by conducting no-box attacks solely using data generated by diffusion models.

Attack through the Input of AI Agent

AI Agent has vulnerabilities in the input link. Attackers can use the diffusion model to generate adversarial samples and destroy the perception ability of the agent by interfering with the input layer data of the agent. So we proposed AdvDiff. Compared with traditional methods (such as PGD gradient injection), AdvDiff proposes two new adversarial guidance techniques to achieve explainable adversarial sampling in the reverse generation process of the diffusion model, which can generate more realistic and high-quality adversarial samples. Experiments show that AdvDiff's attack effect and generation quality on MNIST and ImageNet are better than existing unrestricted adversarial attack methods.

Vulnerabilities in the Cross Agent

There may be vulnerabilities in different parts of a single agent, as well as between multiple agents. Attackers can generate adversarial samples with higher transfer rates to interfere with agents in various scenarios. Therefore, we propose the StyleLess method, which uses a stylized network as a proxy model and perturbs adaptive instance normalization to eliminate style features in adversarial samples, thereby significantly improving the attack transferability of adversarial samples. Experiments show that its transfer attack performance is better than existing methods, and it is compatible with a variety of attack techniques to achieve better results.

Objectives

To construct a practical and secure AI agent system that ensures robust and safe AI agent deployment for commercialization. Our unified, secure AI agent system incorporates our proposed safety-aligned LLM models and secure memory access, and provides legitimate AI agent responses. We plan to implement this system across various open-source AI models, thereby enhancing its security and robustness to be applied in the real world.

Build an AI Agent

Start by constructing an AI agent using our framework. This step guides you through the process of setting up the foundational architecture, configuring the agent's parameters, and integrating it with various platforms. Implement a attack demo

Learn how to implement various attack scenarios to understand potential vulnerabilities in AI agent systems. Understanding these techniques is crucial for building robust defense mechanisms.

Our attack demo framework provides pre-built attack templates and allows you to customize attack parameters. You'll gain hands-on experience in identifying system weaknesses and understanding how malicious actors might exploit AI agents in real-world scenarios.

Test the attack demo

Execute and evaluate the attack demonstrations against your AI agent in a controlled testing environment. This phase allows you to measure the impact of the attack, analyze system responses, and identify critical vulnerabilities.

You can utilize automated testing tools to execute multiple attack scenarios and compare the results. The testing environment is isolated to ensure safe experimentation without affecting production systems.

Deploy/test your defense/attack methods
